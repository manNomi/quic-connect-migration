# Chapter 1 Scanner Trigger Summary

작성일: `2026-06-30`

이 문서는 `tools/scan_implementation_evidence.py`를 공개 QUIC 구현체 shallow clone에 다시 실행한 결과다. 외부 소스 코드는 복사하지 않고, match count와 원본 파일/라인 링크만 남긴다.

## 해석 위치

- 이 표는 mature/immature를 자동 판정하지 않는다.
- `matches`는 keyword hit 수이고, `files`는 해당 category가 발견된 파일 수다.
- 링크는 수동 검토를 시작할 원본 위치다. 실제 판정은 source/test/manual run으로 확인해야 한다.
- Chromium/Cronet, AWS CloudFront, AWS NLB는 전체 source clone 또는 managed service 특성 때문에 이 scanner 표에는 넣지 않고 별도 공식 링크 부록에서 다룬다.

## 해석 주의

- scanner는 keyword 기반 first-pass 도구다. match count가 높다고 maturity가 높다는 뜻은 아니다.
- trigger 위치는 사람이 source/test를 읽기 위한 시작점이다.
- test-only/internal API와 public application API는 구분해야 한다.
- `PATH_CHALLENGE`/`PATH_RESPONSE` 존재는 path validation primitive 근거지만, browser HTTP/3 handover 성공 근거는 아니다.
